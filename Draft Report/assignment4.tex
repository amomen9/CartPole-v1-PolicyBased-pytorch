\begin{abstract}
This article focuses on the implementation of state-of-the-art deep policy-based algorithms PPO and SAC which add additional adjustments to the Loss function of  
policy-based methods of Reinforcement Learning for the CartPole problem. The different methods' performance are compared to each other and to a benchmark solution (baseline) and DQN solution, as well. The implemented methods include first \textbf{REINFORCE}, then attempting to improve its solution by adding some critic to the learning process using \textbf{Actor-Critic} methods \textbf{AC} and \textbf{A2C}.
\end{abstract}

\section{Introduction}\label{sec:introduction}

For problems that practically cannot be addressed by tabular methods—such as CartPole with continuous observations and discrete actions or robotic movement control—algorithms using function approximation is the only viable option. Function approximations can be used to train $Q$-value approximators and/or policy function approximators. If only $Q$-values are approximated, the action space it discrete or discretized, and the policy estimation is performed through $Q$-values, the algorithm is called Deep Q-learning (DQN). If the policy is estimated directly by training a network estimator, the algorithm is a \textbf{policy-based} algorithm. However, the $Q$-values can also be trained as a means and \textbf{critic} to better adjust and improve the policy training. The \textbf{REINFORCE} algorithm purely trains the policy directly, but \textbf{AC} and \textbf{A2C} algorithms blend in value adjustments. This study is dedicated to the implementation and analysis report of these algorithms, in order to quantify how they compare to each other and how they improve on DQN.

In \autoref{sec:theory} the theory necessary for understanding the methods used in this report will be introduced. In \autoref{sec:experiments} the environment, agent and setup of the experiments is explained. The resulting figures are shown in \autoref{sec:results} and discussed in \autoref{sec:discussion}. We conclude our findings in \autoref{sec:conclusion}.

\section{Theory}\label{sec:theory}
\subsection{REINFORCE} %just as a reference, can be removed later
\vspace{-0.2cm}
\subsubsection{Vanilla loss}
\vspace{-0.1cm}
The term ‘vanilla loss’ is more commonly attributed to the REINFORCE algorithm. When we consider the loss function to be in the simplest form:
$$L = -\mathbb{E}[\log \pi_{\theta}(a \mid s) \cdot R]$$
where 
$$R=G_t=\sum_{k=0}^{\infty} \gamma^k r_{t+k}$$

is the raw Monte-Carlo return. It is the basic version without extra stabilizers or modifications, such as clipping as in PPO, trust-region constraints as in TRPO or penalties, entropy bonuses,
and sophisticated off-policy corrections. 
It was introduced to optimize a policy \textbf{directly}, handle \textbf{continuous action spaces} naturally, and avoid needing to differentiate through the environment.
\vspace{-0.2cm}
\subsubsection{Algorithm}
\vspace{-0.1cm}
The algorithm for the plain REINFORCE approach is shown in \autoref{alg:REINFORCE}. We have multiple full-episodes (in Monte-Carlo manner) that we use to train our estimation function with. For CartPole, we have to sample episodes that end with falling of the pole (trespassing maximum deviation from a vertical position), and we feed the system with an episode, one at a loop iteration to adjust the training via gradient ascend/descend method. The learning rate $\alpha$ is used to regulate the policy update. To find the best policy the parameters $\theta$ must be updated such that the return is maximised. 
% \todo{Include derivation of vanilla loss}

\begin{algorithm}[H]
    \caption{REINFORCE (Monte Carlo Policy Gradient)}
    \begin{algorithmic}
        \STATE \textbf{Input:} policy network $\pi_\theta(a \mid s)$, number of episodes $M$,
        learning rate $\alpha$, discount $\gamma$
        \STATE \textbf{Initialise:} Randomly initialise $\theta$
        \FOR{$i \in 1, \ldots, M$}
        \STATE Sample trace $\tau = \{s_0, a_0, r_0, s_1, \ldots, s_T\}$
        \FOR{$t \in 0, \ldots, T - 1$}
        \STATE $\hat{Q}(s_t,a_t)= \sum_{k=0}^{T-t-1} \gamma^k r_{t+k}$
        \ENDFOR
        \STATE $\theta \leftarrow \theta + \alpha \cdot \sum_t \hat{Q}(s_t,a_t) \cdot \nabla_\theta \log[\pi_\theta(a_t \mid s_t)]$
        \ENDFOR
    \end{algorithmic}
    \label{alg:REINFORCE}
\end{algorithm}

\subsection{AC} %just as a reference, can be removed later
\vspace{-0.1cm}
The REINFORCE algorithm can be expanded by introducing a neural network for the value function $V_\varphi(s)$ that needs to be estimated using the same Monte-Carlo estimation. The reason for using a value function network $V_\varphi(s)$ instead of a state-action value network $Q_\varphi(s)$ is mostly simplicity and this helps the code be similar to the code for the A2C algorithm (which will be explained later). The policy and value networks are referred to as the actor and the critic respectively. Using this actor-critic (AC) approach utilises the low-bias aspect from REINFORCE due to the random action selection and reduces REINFORCE's high-variance by introducing the critic \cite{RL_book}.

Before, the DQN algorithm was applied to the CartPole environment. This off-policy algorithm used the $\max_{a'}Q(s',a')$ operation to help determine Bellman target that is used in the loss that needed to be minimised. This same loss cannot be used for the AC algorithm however. For this algorithm actions are chosen based on the probability assigned to them by the current policy $\pi_\theta$. The AC algorithm aims to approximate the value function under the current stochastic policy, not under the optimal policy, which is what the DQN algorithm does. The loss function that is now used to update the actor network weights is

\vspace{-0.4cm}
\begin{equation*}
    L(\theta) = -\sum_t \hat{Q}(s_t,a_t) \log[\pi_\theta(a_t|s_t)]
    \vspace{-0.4cm}
\end{equation*}

Where $\hat{Q}(s_t,a_t)$ is the Monte-Carlo estimate for $Q^{\pi_\theta}(s_t,a_t)$. The loss function that is used to update the critic network weights is
\vspace{-0.1cm}
\begin{equation*}
    L(\varphi) = \sum_t (\hat{Q}(s_t,a_t) - V_\varphi(s_t))^2
    \vspace{-0.2cm}
\end{equation*}
Using $V_\varphi(s)$ instead of $Q_\varphi(s,a)$ might seem unintuitive. However, since the two are related via
\vspace{-0.1cm}
\begin{equation*}
V^{\pi_\theta}(s)=\mathds{E}_{a\sim\pi_\theta}[Q^{\pi_\theta}(s,a)]
\vspace{-0.1cm}
\end{equation*}
using $Q_\varphi(s,a)$ instead would result in approximately the same quantity being minimised. This is only the case since the loss is calculated over many episodes. The pseudocode for this algorithm is shown in \autoref{alg:AC}.
\vspace{-0.2cm}
\begin{algorithm}[H]
    \caption{Actor Critic (AC) algorithm}
    \begin{algorithmic}
        \STATE \textbf{Input:} policy network $\pi_\theta(a|s)$ (actor), value network $V_\varphi(s)$ (critic), number of episodes $M$, actor learning rate $\alpha$, critic learning rate $\beta$ and discount factor $\gamma$
        \STATE \textbf{Initialise:} Randomly initialise $\theta$ and $\varphi$
        \FOR{$i\in 1,...,M$}
        \STATE Sample trace $\tau=\{s_0,a_0,r_0,s_1,...,s_T\}$
        \FOR{$t \in 0,...,T-1$}
        \STATE $\hat{Q}(s_t,a_t)=\sum_{k=0}^{T-t-1} \gamma^k\cdot r_{t+k}$
        \ENDFOR
        \STATE $\theta\leftarrow\theta+\alpha\cdot\sum_t \hat{Q}(s_t,a_t) \cdot \nabla_\theta \log[\pi_\theta(a_t|s_t)]$
        \STATE $\varphi \leftarrow \varphi - \beta \cdot \sum_t (\hat{Q}(s_t,a_t) - V_\varphi(s_t))^2$
        \ENDFOR
    \end{algorithmic}
    \label{alg:AC}
\end{algorithm}

\subsection{A2C} %just as a reference, can be removed later

The Actor-Critic (AC) approach described above uses a full Monte-Carlo return, taking all the rewards into account until the end of an episode. In contrast to AC, Advantage Actor-Critic (A2C) computes the $n$-step target. It sums over $n$ rewards and then bootstraps using the critic. This is also the reason why the AC algorithm doesn't include the estimation depth term used in determining $\hat{Q}(s_t,a_t)$. By calculating the Monte-Carlo target, instead of an $n$-step target, the value of $V_\varphi(s_{t+n})$ equals zero, since $s_{t+n}$ is the terminal state.

Furthermore, while AC updates the actor directly with the target $\hat{Q}$, A2C subtracts the baseline (the value function) to compute the advantage. The advantage represents the value of taking a specific action in a certain state relative to the value of that state, thereby isolating the contribution of the action from the overall quality of the state. This advantage is then used to update the actor, which reduces the variance of the policy gradient updates. The full algorithm for A2C is showed in \autoref{alg:A2C}


\begin{algorithm}[H]
    \caption{Advantage Actor Critic (A2C) algorithm}
    \begin{algorithmic}
        \STATE \textbf{Input:} policy network $\pi_\theta(a|s)$ (actor), value network $V_\varphi(s)$ (critic), number of episodes $M$, estimation depth $\tilde{n}$, actor learning rate $\alpha$ and critic learning rate $\beta$
        \STATE \textbf{Initialise:} Randomly initialise $\theta$ and $\varphi$
        \WHILE{not converged}
        \FOR{$i\in 1,...,M$}
        \STATE Sample trace $\tau=\{s_0,a_0,r_0,s_1,...,s_T\}$
        \FOR{$t \in 0,...,T-1$}
        \STATE $\hat{Q}_n(s_t,a_t)=\sum_{k=0}^{n-1} \gamma^k\cdot r_{t+k}+\gamma^{\tilde{n}}\cdot V_\varphi(s_{t+n})$
        \STATE $\hat{A}_n(s_t,a_t)=\hat{Q}_n(s_t,a_t)-V_\varphi(s_t))$
        \ENDFOR
        \STATE $\theta\leftarrow\theta+\alpha\cdot\sum_t \hat{A}_n(s_t,a_t) \cdot \nabla_\theta \log[\pi_\theta(a_t|s_t)]$
        \STATE $\varphi \leftarrow \varphi - \beta \cdot \sum_t (\hat{A}_n(s_t,a_t))^2$
        \ENDFOR
        \ENDWHILE
    \end{algorithmic}
    \label{alg:A2C}
\end{algorithm}

\section{Experiments}\label{sec:experiments}

In order to test different algorithms, the CartPole environment from the open-source Python library Gymnasium \cite{Towers2024} is used. Gymnasium provides a standardized Application Programming Interface (API) for developing and testing different RL algorithms and includes several physics simulation environments. The CartPole environment is two-dimensional and consists of a pole placed vertically on a cart with a passive joint. The cart has the ability to move sideways along a frictionless track. The pole acts like a pendulum, with the goal of balancing it for as long as possible by applying forces in the left and right direction onto the cart. 

The application of the left and right forces are the two actions that can be taken in the environment and together form the action space. The observation space consists of four observables: The cart position, the cart velocity, the pole angle and the pole angular velocity. An episode is terminated if the angle of the pole is more than $12^{\circ}$ or if the position of the cart leaves the $(-2.4, 2.4)$ range. A reward of 1 is given for every timestep that the pole remains upright, with a threshold of 500, due to a maximum of 500 timesteps per episode.

Each algorithm is run for one million timesteps. To get a reliable estimate of the performance, each of the learning curves are averaged over five repetitions. Furthermore, their Gaussian standard deviation is calculated to account for the uncertainty margin.

\vspace{-0.2cm}
\subsection{A2C} %just as a reference, can be removed later
\vspace{-0.1cm}

The hyperparameters that are considered to have the greatest effect on the learning process are tuned to result in an optimal learning curve. The first one is the estimation depth $\tilde{n}$, which dictates the number future rewards that are considered in the $n$-step target, before bootstrapping. Since the policy (actor) and the value function (critic) are each estimated by a separate neural network (NN). They each have there own learning rate and network architecture, which is the number and size of hidden layers. So the policy learning rate, the value function learning rate, the hidden layers of the policy NN, and the hidden layers of the value function NN are also hyperparameters that need to be optimized.

These five hyperparameters were optimized using a systematic approach, where one hyperparameter is changed to different values while the others are kept at a constant value. The performance of the different values for the variable hyperparameters after a million timesteps are then compared to each other to determine which one results in the most optimal and stable learning curve. These comparisons result in the optimal version of the A2C (see \autoref{fig:main_result}), which has $\tilde{n}=10$, $\alpha=0.0001$, $\beta=0.01$, a policy architecture of two hidden layers with 64 neurons each, and a value function architecture of two hidden layers with 128 neurons each.

\section{Discussion}\label{sec:discussion}



\vspace{-0.2cm}
\subsection{Improvements from DQN}
\vspace{-0.1cm}
The DQN algorithm had a network be a part of both the predicted value and the target value used to determine the loss. In order to increase stability and break temporal correlations between states two improvements were made using a target network and replay buffer respectively. These improvements cannot be applied to REINFORCE, AC or A2C however. The training stability that the target network provides for DQN is not necessary in the algorithms covered here since the target and prediction used in the loss do not both depend on the same $Q$-network. This \textit{is} the case for DQN. 

The replay buffer used actions taken from previous versions of the policy, which worked when the optimal policy needed to be approximated. In the algorithms covered in this work however the current policy $\pi_\theta$ needed to be approximated. The replay buffer works for off-policy algorithms, like DQN, but not for on-policy algorithms.

\vspace{-0.2cm}
\subsection{REINFORCE} %just as a reference, can be removed later
\vspace{-0.1cm}

Compared to baseline and DQN, this algorithm performs better at earlier timesteps. As it reaches higher timesteps, the performance becomes similar to the baseline but excels that of the DQN, and unlike DQN, it does not have a adverse training misbehaviour as the training continues, but it appears that training can be enough with roughly 300,000 time steps. This algorithm performs above expectations, given its simplicity.


\vspace{-0.2cm}
\subsection{AC} %just as a reference, can be removed later
\vspace{-0.1cm}

Compared to the baseline data, the AC algorithm performs well at earlier timesteps. As it reaches towards higher timesteps, the performance is similar to that of the baseline data. At these earlier timesteps, the episodes are relatively short which leads to noisy estimates of the gradient. This explains the larger variance seen at these smaller timesteps. The stability of the performance seems to improve as the algorithm is kept running. This is recognised by the reducing frequency of sudden drops in the episode return (which are still seen in the baseline data). For the AC algorithm the evaluation interval is ten times as large as the one used in REINFORCE and A2C. This was an optimisation used to help reduce the runtime in order to reach the one million timesteps. However, this has come at the cost of loss of information.

Compared to DQN, the AC algorithm performs better, since its return keeps increasing after 400,000 timesteps, whilst the DQN's episode return starts to trend downwards. The AC algorithm performs similar to REINFORCE, but starts at a higher episode return in its earlier timesteps. However it does not have the stability that REINFORCE or A2C has, as some dips in the episode return are observed. This can also be an artifact due to the higher evaluation interval and using the same smoothing window. To better compare the algorithms, ideally the same evaluation interval is used.


\vspace{-0.2cm}
\subsection{A2C} %just as a reference, can be removed later
\vspace{-0.1cm}

% The best hyperparameters for A2C mentioned in \autoref{sec:results} were determined by comparing the learning curves for different values of the concerning hyperparameter to each other. For the estimation depth, the policy learning rate, and the value function architecture the chosen values are were clearly the best performing values after one million timesteps. For the value function learning rate and the policy architecture the distinction between the tested values was less clear. In the case of the value function learning rate, the values of 0.01 and 0.001 both perform roughly equally well, while a learning rate of 0.0001 performs significantly worse. However, 0.01 seemed to have a somewhat lower error margin, suggesting a more stable learning process, even though the difference is small. For the policy architecture, two hidden layers with 128 neurons each results in a very unstable learning curve over many timesteps, with an episode return that spikes and dips without converging. From the two remaining architectures, the one with two hidden layers of 64 neurons each seems to perform consistently better that the one with 1 hidden layer of 32 neurons. The figures of all the learning curves can be found in \autoref{app:a2c_hyper} for further inspection.

The resulting learning curve of an optimally tuned A2C agent performs consistently better than DQN and the baseline (that is generally considered to solve the CartPole problem in a reasonable amount of timesteps), as can be seen in \autoref{fig:main_result}. The A2C learning curve quickly rises well above DQN and the baseline and reaches the optimum return of 500 after about 400,000 timesteps within the uncertainty margin. Then it maintains this optimum with relatively low standard deviation, outperforming the other algorithms. However, before it converges to the optimum, this error margin is significantly larger. This could be improved by averaging over more repetitions that five, but this would come at the cost of an increased computation time.

Follow-up studies could improve A2C further by introducing multiple parallel agents (A3C), constraining the policy change in a single update (PPO) or adding an entropy term to the reward (SAC). 

\section{Conclusion}\label{sec:conclusion}
With the algorithms covered in this work a comparison can be made between value-based DQN and policy-based REINFORCE, AC and A2C. All three policy-based algorithms out-perform DQN. Whilst REINFORCE and AC both have dips in their return, A2C's performance is almost constantly increasing and stable. Using these algorithms has shown how relatively simple algorithms can outperform the more complicated DQN algorithm, even with the added tricks to improve its episode return.

\section{Hyperparameter Optimization A2C}\label{app:a2c_hyper}
The estimation depth $\tilde{n}$ was tested for the values 1, 5, and 10, the policy learning rate $\alpha$ was tested for the values 0.001, 0.0001 and 0.00001 and the value function learning rate $\beta$ was tested for the values 0.01, 0.001 and 0.0001. The hidden layer architectures used for optimization were [32], [64, 64] and [128, 128] for both neural networks.\\
The hyperparameters that remained unvaried during these experiments were an estimation depth $\tilde{n}$ of 5, a learning rate ($\alpha$ and $\beta$) of 0.0001 and a hidden layers architecture of [64, 64] for both the policy and the value function neural networks

For the estimation depth, the policy learning rate (see \autoref{fig:a2c_lr_policy}), and the value function architecture (see \autoref{fig:a2c_hl_value_function}) the chosen values are clearly the best performing values after one million timesteps. For the value function learning (see \autoref{fig:a2c_lr_value_function}) rate and the policy architecture (see \autoref{fig:a2c_hl_policy}) the distinction between the tested values was less clear. In the case of the value function learning rate, the values of 0.01 and 0.001 both perform roughly equally well, while a learning rate of 0.0001 performs significantly worse. However, 0.01 seemed to have a somewhat lower error margin, suggesting a more stable learning process, even though the difference is small. For the policy architecture, two hidden layers with 128 neurons each results in a very unstable learning curve over many timesteps, with an episode return that spikes and dips without converging. From the two remaining architectures, the one with two hidden layers of 64 neurons each seems to perform consistently better that the one with one hidden layer of 32 neurons.
